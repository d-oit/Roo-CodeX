# Architecture Plan

## 1. Event Storming

### Commands

- **`SubmitUserPrompt`**: A user submits a natural language query or instruction to the chat interface.

    - _Parameters_: `promptText` (string), `context` (object)
    - _Triggers Event_: `UserPromptSubmitted`

- **`RequestCodeAnalysis`**: A user explicitly asks for an analysis of the current codebase.

    - _Parameters_: `scope` (string - e.g., "file", "workspace")
    - _Triggers Event_: `CodeAnalysisRequested`

- **`ApplyCodeSuggestion`**: A user accepts a code suggestion provided by the AI.

    - _Parameters_: `suggestionId` (string), `filePath` (string), `codeBlock` (string)
    - _Triggers Event_: `CodeSuggestionApplied`

- **`SwitchToAgentMode`**: A user activates the autonomous agent mode for multi-step tasks.
    - _Parameters_: `taskDescription` (string)
    - _Triggers Event_: `AgentModeActivated`

### Events

- **`UserPromptSubmitted`**: Fired when a user sends a message.

    - _Payload_: `promptText`, `context`, `timestamp`
    - _Updates Views_: `ChatHistoryView`

- **`CodeAnalysisRequested`**: Fired when a user requests a codebase analysis.

    - _Payload_: `scope`, `timestamp`

- **`CodeSuggestionProvided`**: Fired when the AI generates a code suggestion.

    - _Payload_: `suggestionId`, `filePath`, `codeBlock`, `explanation`
    - _Updates Views_: `SuggestionsView`

- **`CodeSuggestionApplied`**: Fired when a user applies a suggestion.

    - _Payload_: `suggestionId`, `filePath`
    - _Updates Views_: `EditorView`, `ChatHistoryView`

- **`AgentModeActivated`**: Fired when agent mode is started.

    - _Payload_: `taskId`, `taskDescription`
    - _Updates Views_: `AgentStatusView`

- **`AgentTaskProgress`**: Fired as the agent completes steps.
    - _Payload_: `taskId`, `step`, `status`, `output`
    - _Updates Views_: `AgentStatusView`, `TerminalView`

## 2. View Design

### View: `ChatHistoryView`

- **Description:** Displays the conversation history between the user and the AI.
- **Schema:**
    - `messages`: (array)
        - `sender`: (string - "user" or "ai")
        - `content`: (string)
        - `timestamp`: (datetime)
- **Updated by Events:** `UserPromptSubmitted`, `CodeSuggestionApplied`

### View: `SuggestionsView`

- **Description:** Shows the code suggestions generated by the AI.
- **Schema:**
    - `suggestions`: (array)
        - `suggestionId`: (string)
        - `filePath`: (string)
        - `codeBlock`: (string)
        - `explanation`: (string)
- **Updated by Events:** `CodeSuggestionProvided`

### View: `AgentStatusView`

- **Description:** Provides real-time status of the autonomous agent's tasks.
- **Schema:**
    - `agentActive`: (boolean)
    - `currentTask`: (string)
    - `progress`: (array)
        - `step`: (string)
        - `status`: (string)
- **Updated by Events:** `AgentModeActivated`, `AgentTaskProgress`

### View: `EditorView`

- **Description:** Represents the state of the code editor, which is updated when suggestions are applied.
- **Schema:**
    - `openFile`: (string)
    - `fileContent`: (string)
- **Updated by Events:** `CodeSuggestionApplied`

### View: `TerminalView`

- **Description:** Displays output from agent-executed commands.
- **Schema:**
    - `outputLines`: (array of strings)
- **Updated by Events:** `AgentTaskProgress`

## 3. API Specification

### Endpoint: `POST /api/chat/prompt`

- **Description:** Submits a user prompt to the AI.
- **Request Body:**
    - `promptText`: (string)
    - `context`: (object)
- **Responses:**
    - `202 Accepted`: The prompt has been accepted for processing.

### Endpoint: `POST /api/suggestions/apply`

- **Description:** Applies a given code suggestion to a file.
- **Request Body:**
    - `suggestionId`: (string)
- **Responses:**
    - `200 OK`: Suggestion applied successfully.
    - `404 Not Found`: Suggestion not found.

### Endpoint: `POST /api/agent/activate`

- **Description:** Activates the agent mode with a specific task.
- **Request Body:**
    - `taskDescription`: (string)
- **Responses:**
    - `202 Accepted`: Agent mode activated.

### Endpoint: `GET /api/views/chat-history`

- **Description:** Retrieves the `ChatHistoryView`.
- **Responses:**
    - `200 OK`: Returns the chat history view object.

### Endpoint: `GET /api/views/suggestions`

- **Description:** Retrieves the `SuggestionsView`.
- **Responses:**
    - `200 OK`: Returns the suggestions view object.

### Endpoint: `GET /api/views/agent-status`

- **Description:** Retrieves the `AgentStatusView`.
- **Responses:**
    - `200 OK`: Returns the agent status view object.

## 4. Implementation Plan

### Technology Stack

- **Frontend:** VS Code Extension API, TypeScript, React (for webviews)
- **Backend (Command/Event Processing):** Node.js, TypeScript
- **Event Store:** A simple in-memory store for this phase, potentially moving to a persistent store like a file or a database later.

### File Structure

```
src/
├── extension.ts        // Main extension entry point
├── commands/           // Handlers for user commands
│   ├── submitPrompt.ts
│   └── applySuggestion.ts
├── events/             // Event definitions and processors
│   ├── userPromptSubmitted.ts
│   └── codeSuggestionApplied.ts
├── views/              // Logic for building and updating views
│   ├── chatHistoryView.ts
│   └── suggestionsView.ts
├── webviews/           // React components for the UI
│   ├── chat/
│   └── agent/
└── store/              // In-memory event store
```

### Key Components

- **Command Bus:** A central dispatcher for all incoming commands.
- **Event Bus:** A central dispatcher for all system events.
- **View Projectors:** Components that listen for events and update the corresponding read models (views).

### Testing Strategy

- **Unit Tests:** For individual commands, event processors, and view projectors.
- **Integration Tests:** To verify the flow from command to event to view update.
- **E2E Tests:** Using the VS Code test framework to simulate user interactions.

### Deployment Strategy

- **Packaging:** Use `vsce` to package the extension.
- **Distribution:** Publish to the Visual Studio Code Marketplace.
